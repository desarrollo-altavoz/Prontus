Variable Valor
ARTIC_ACTUALIZA_PORTS
Habilita o deshabilita la actualización de todas las portadas donde está publicado un artículo, al guardarlo. 		   
ACTUALIZACIONES
Permite habilitar o deshabilitar las actualizaciones automáticas. 		   
PUBLIC_SERVER_NAME
Se debe indicar el nombre del servidor, necesario para la ejecución de los crons.
SERVER_SMTP
Indica el servidor SMTP a utilizar en el sitio Prontus.
CONTROL_FECHA
Activa o desactiva la opción para controlar la fecha y hora de publicación y de expiración de artículos, lo cual permite que las portadas se actualicen según estos valores al ser guardadas manualmente o mediante un proceso cron. 		   
CONTROLAR_ALTA_ARTICULOS
Activa o desactiva el control de la disponibilidad de un artículo para ser incluido en portadas, artículos relacionados, portadas taxonómicas y buscador. 		   
COMPORTAMIENTO_ALTA_ARTICULOS
Define qué comportamiento tendrán las vistas de los artículos sin alta.
ABRIR_FIDS_EN_POP
Define si los artículos se abrirán en una popup como en el sistema antiguo. 		   
USAR_PUBLIC_SERVER_NAME_VER_ARTIC
Define si al hacer click en "Ver Artículo" desde el FID se usará la ruta absoluta del artículo usando el nombre del servidor (PUBLIC_SERVER_NAME).
Por ejemplo:
http://%%PUBLIC_SERVER_NAME%%/%%_PRONTUS_ID%%/site/artic/...
Por defecto se utiliza la ruta relativa. 		   
COMENTARIOS
Habilita o deshabilita el sistema de comentarios. 		   
VTXT_PASTE_NEWLINES_AS_P
Habilita o deshabilita la transformación de saltos de linea a <p>. 		   
VTXT_DTD
Permite definir el uso de xhtml 1.0 strict o transitional para el código generado en el VTXT.
VTXT_ENCODE_CHARS
Hace un encode de los caracteres sensibles. Es el responsable de escapear los tags HTML colocando espacios entre los signos < y >.
Ayuda: Configurar en NO, si se requiere que el VTXT parsee código (por ejemplo en manuales o ejemplos) 		   
VTXT_MEDIA_SCRIPT
Indica si se usará el embed tradicional o JavaScript.
Al habilitarlo, las etiquetas <object> se convierten a código JavaScript, por ejemplo: writeFlash(...) 		   
FRIENDLY_URLS
Habilita o deshabilita las url amigables. 		   
FRIENDLY_URLS_VERSION
Especifica la versión de las url amigables que se utilizará.
De la forma: /titular/prontus_id/aaaa-mm-dd/hhnnss.ext


De la forma: /prontus_id/seccion/tema/subtema/titular/aaaa-mm-dd/hhnnss.ext. En caso de que el artículo no tenga sección, tema o subtema, se omiten y no se colocan en la url.


Utiliza el nombre de dominio o subdominio para identificar la instancia de Prontus. Por ejemplo:

- http://www.<prontus_id>.cl/seccion/tema/subtema/titular/aaaa-mm-dd/hhnnss.ext
- http://www.<prontus_id>.dominio.cl/seccion/tema/subtema/titular/aaaa-mm-dd/hhnnss.ext

Nota: El prontus_id no puede tener guión bajo (_).
FRIENDLY_URLS_LARGO_TITULAR
Define el largo máximo que tendrá el titular en las Friendly URL. Se recomienda no exceder los 75 caracteres.
FRIENDLY_URL_IMAGES
Habilita o deshabilita las url amigables para imágenes. 		   
MULTIVISTAS
Define las vistas que tendrá Prontus, además de la vista por defecto (vista principal).
Nombre Quitar
+ Agregar
UPLOADS_PERMITIDOS
Especifica que tipo de documentos (extensiones) estarán permitidas al momento de subir archivos en los fids. 		%%UPLOADS_PERMITIDOS%%
Cambiar
SERVER_PROTOCOLO_HTTPS
Indica si se usará un servidor https tanto en el Frontend, como en el Backend. 		   
VARNISH_SERVER_NAME
Define los servidores de Varnish Cache.
+ Agregar
VARNISH_GLOBAL_PURGE
Indicar las rutas serán expiradas cada vez que se haga purge. Se recomienda indicar sólo lo necesario para no sobrecargar el sistema. Una ruta por línea (no debe incluír el dominio).

Ejemplos:
/prontus/site/edic/base/port/portada.html
/noticias/
/
FORM_CSV_CHARSET
Define el charset del archivo de Respaldo del Prontus Form.
Si al importar dicho archivo, aparecen caracteres extraños, se recomienda probar otro valor.
POST_PROCESO de borrado de artículos
El proceso o script que indique, se ejecutará cada vez que se borre un artículo.
Para que sea un script válido, debe ubicarse en el mismo dir. de CGIs de Prontus, o a lo más, un nivel hacia arriba.
SCRIPT_QUOTA
Script para el cálculo de quota. Puede ser PHP o Perl y será invocado desde la raíz de sitio.
Ejemplo: /cgi-cpn/mi_script_quota.pl
FOTO_MAX_PIXEL
Se indica el tamaño máximo en pixels que tendrán las fotos al ser subidas al banco de imágenes. De tal forma que si una foto supera el máximo, esta será redimensionada para estar por debajo del limite establecido, por ejemplo: 1920x1080

Si se deja en blanco, no habrá límite.
BLOQUEO_EDICION
Especifica la modalidad de bloqueo de edición de artículos y portadas.
Solo advertencia.


Advertencia con prompt e ingreso de número aleatorio de 4 dígitos.


Bloqueo total. El artículo solo será editable por la primera persona que lo abrió.
DROPBOX
Activa o desactiva el respaldo de archivos en Dropbox. 		   
CLOUDFLARE
Activa o desactiva la opción de hacer purge de contenido prontus en CloudFlare. Al activar esta opción, Varnish quedará deshabilitado. 		   
TRANSCODIFICACIÓN
Variable Valor
MAX_XCODING
Tamaño máximo (en MB) para la transcodificación de videos.
DIR_FFMPEG
Indica la ruta a la carpeta donde se encuentra el ejecutable de ffmpeg.

Verificar Directorio
FFMPEG_PARAMS
Parámetros que se utilizarán con el comando ffmpeg en la transcodificación de videos.
USAR_LIB_FDK
Activa el uso de la librería libfdk_aac.

Requiere FFMPEG con libfdk_aac instalada. Si no se encuentra disponible, la transcodificación fallará
 		   
ADVANCED_XCODING
Activa las opciones avanzadas de transcodificación.
Las opciones de configuración avanzada aparecerán bajo el ítem del menú "Transcodificación +"

Requiere FFMPEG version 1 o mayor
